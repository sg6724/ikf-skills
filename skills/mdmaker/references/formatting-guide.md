# Markdown Formatting Guide

## Headers
Use `#` for headers. More `#` = smaller header.

```markdown
# H1 - Main Title
## H2 - Section
### H3 - Subsection
```

## Links
Create clickable links with `[text](url)`:

```markdown
[Google](https://google.com)
[About Page](/about)
```

## Lists
```markdown
- Bullet item
- Another item

1. Numbered item
2. Another item
```

## Emphasis
```markdown
**Bold text**
*Italic text*
`inline code`
```

## Tables
```markdown
| Header 1 | Header 2 |
|----------|----------|
| Cell 1   | Cell 2   |
```

## Code Blocks
Use triple backticks with optional language:

```python
def example():
    return "Hello"
```

## Best Practices

1. **Hierarchy**: Use headers to create clear document structure
2. **Links**: Always use absolute URLs for external links
3. **Readability**: Keep lines reasonably short
4. **Consistency**: Use consistent formatting throughout
